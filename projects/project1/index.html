Deployable Site

FastAPI backend, plain HTML/CSS/JS frontend. No build step.

active

What it is

A web starter that deploys anywhere. FastAPI for the API. Hand-written HTML/CSS/JS for the page. No webpack, no React, no build. The whole thing is one directory.

Stack

LayerTechWhy
BackendFastAPI + UvicornAsync, typed, OpenAPI docs
TemplatingJinja2Server HTML, shared layouts
StylingVanilla CSSCustom properties, no preprocessor
FrontendVanilla JSNative fetch, no framework
DeployRailway / Render / Fly.ioReads PORT env, one command start

File tree

deployable-site/
├── app.py              # FastAPI app
├── requirements.txt    # deps
├── railway.json        # Railway config
├── templates/
│   └── index.html      # Jinja2 template
└── static/
    ├── style.css       # design system
    └── script.js       # contact form handler

Endpoints

MethodPathDescription
GET/Landing page
GET/api/healthHealth check JSON
GET/api/nowCurrent UTC time
POST/api/contactContact form

Deploy commands

Railway

railway login
railway init
railway up

Render

render services create \
  --name deployable-site \
  --type web_service \
  --runtime python \
  --build-command "pip install -r requirements.txt" \
  --start-command "uvicorn app:app --host 0.0.0.0 --port \$PORT"

Local

pip install -r requirements.txt
uvicorn app:app --reload
Static version at anthonyfangqing.github.io/deployable-site. Full backend needs a Python host.

Notes

  • StaticFiles with html=True so subdirectories work.
  • SPA fallback returns the landing page for unknown routes.
  • Starlette changed the TemplateResponse signature: pass request first, not in the context dict.
  • Railway free tier blocks deploys 8 AM – 8 PM CEST. Render wants a card even for free tier.
