Discord Gateway Config

Free-response channels, mention rules, and auto-threading for the Hermes Discord adapter.

active

Context

Hermes connects to Discord via the DiscordAdapter plugin. By default, the bot only responds when explicitly @mentioned in server channels. This is sane default behavior, but for some channels you want the bot to reply to every message — no mention required.

What was configured

Three mechanisms control how Hermes listens on Discord:

  • Require mention — global toggle. If true, messages in server channels need an @bot mention. DMs and known threads are exempt.
  • Free response channels — allowlist of channel IDs where mention is skipped even if require-mention is globally enabled.
  • Auto-threading — when enabled, the bot creates a thread per conversation in free-response channels so replies stay scoped.

Config options

Variable / KeyTypeDefaultEffect
DISCORD_REQUIRE_MENTIONbooltrueIf true, server messages need @mention
DISCORD_FREE_RESPONSE_CHANNELSstr/list[]Channel IDs where mention is waived
DISCORD_AUTO_THREADbooltrueAuto-create thread per conversation
DISCORD_ALLOWED_USERSstr"*"User IDs allowed to DM the bot

YAML config example

discord:
  extra:
    require_mention: false
    free_response_channels:
      - "123456789012345678"
      - "987654321098765432"
    auto_thread: true

Code flow

When a message arrives, _handle_message runs these checks in order:

  1. Bot mention filter — if other bots are mentioned but not this one, drop the message.
  2. Ignore-no-mention logic — if the message mentions humans but not this bot, drop unless in a free-response channel.
  3. Free-response check — if the channel ID (or its parent forum ID) is in the allowlist, skip the mention requirement.
  4. Thread tracking — if the bot has already participated in this thread, replies don't need mentions.
  5. Auto-thread — if enabled and the channel is a text channel, spawn a thread and route replies there.
Edge case: YAML parses bare integers as int, not str. The adapter coerces scalar values with str(raw) so free_response_channels: 1491973769726791812 works without quotes.
