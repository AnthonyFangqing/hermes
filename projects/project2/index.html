Discord Gateway Config

Making the bot reply without @mentions in some channels.

active

Context

Hermes connects to Discord via the DiscordAdapter plugin. By default it only answers when @mentioned in server channels. I wanted it to reply to everything in specific channels.

What was configured

Three toggles control how Hermes listens:

  • Require mention: global toggle. When on, server messages need an @bot. DMs and known threads are exempt.
  • Free response channels: list of channel IDs where the mention rule is skipped.
  • Auto-threading: the bot spawns a thread per conversation in free-response channels so replies do not clutter the main channel.

Config options

Variable / KeyTypeDefaultEffect
DISCORD_REQUIRE_MENTIONbooltrueServer messages need @mention
DISCORD_FREE_RESPONSE_CHANNELSstr/list[]Channel IDs where mention is waived
DISCORD_AUTO_THREADbooltrueAuto-create thread per conversation
DISCORD_ALLOWED_USERSstr"*"Users allowed to DM the bot

YAML config example

discord:
  extra:
    require_mention: false
    free_response_channels:
      - "123456789012345678"
      - "987654321098765432"
    auto_thread: true

Code flow

When a message arrives, _handle_message runs these checks:

  1. Bot mention filter: if other bots are mentioned but not this one, drop the message.
  2. Ignore-no-mention: if the message mentions humans but not this bot, drop unless the channel is in the free-response list.
  3. Free-response check: if the channel ID (or its parent forum ID) is in the list, skip the mention rule.
  4. Thread tracking: if the bot already replied in this thread, replies do not need mentions.
  5. Auto-thread: if enabled and the channel is text, spawn a thread and route replies there.
YAML parses bare integers as int, not str. The adapter coerces scalar values with str(raw) so free_response_channels: 1491973769726791812 works without quotes.
