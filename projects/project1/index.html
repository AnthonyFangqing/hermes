Deployable Site

FastAPI starter with a modern dark-themed frontend. Zero build step, one-click deploy.

active

What it is

A minimal but production-ready web starter. Backend is FastAPI (async, typed, auto-documented). Frontend is hand-written HTML/CSS/JS — no webpack, no node_modules, no build pipeline. The entire stack fits in a single directory and deploys to any Python host in under a minute.

Stack

LayerTechWhy
BackendFastAPI + UvicornAsync, OpenAPI docs, dependency injection
TemplatingJinja2Server-rendered HTML with shared layouts
StylingVanilla CSSCSS custom properties, no preprocessor
FrontendVanilla JSNative fetch, no framework weight
DeployRailway / Render / Fly.ioReads PORT env, single command start

Deploy commands

Railway

railway login
railway init
railway up

Render

render services create \
  --name deployable-site \
  --type web_service \
  --runtime python \
  --build-command "pip install -r requirements.txt" \
  --start-command "uvicorn app:app --host 0.0.0.0 --port \$PORT"

Local

pip install -r requirements.txt
uvicorn app:app --reload
Live demo: Static version at anthonyfangqing.github.io/deployable-site. Full backend requires a Python host.

Notes

  • Static asset serving uses StaticFiles with html=True so subdirectories work.
  • SPA fallback catches unknown routes and returns the landing page.
  • Jinja2 TemplateResponse signature changed in newer Starlette — pass request as first positional arg, not in the context dict.
  • Railway free tier blocks deploys 8 AM – 8 PM CEST. Render requires a card on file even for free tier.
